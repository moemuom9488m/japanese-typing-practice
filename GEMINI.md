# Japanese Typing Practice System - AI Agent Guidelines

Hello Agent. You are assisting Ibu in developing and maintaining the "Japanese Typing Practice System". 
We have implemented a **Separation of Concerns** architecture. Please read the following rules carefully before making any modifications.

## 1. Project Architecture
* **`src/App.jsx`**: Contains ALL the core React logic, UI components, Tailwind styling, Canvas particle background, AI API integrations (Gemini Text/TTS), and state management.
* **`src/data.js`**: Acts as the database. Contains the Kanji dictionary (`KANJI_DICT`), Quiz Data (`QUIZ_DATA`), Chapters (`CHAPTERS`), and Practice Types (`PRACTICE_TYPES`).

## 2. Critical Mechanisms (DO NOT BREAK)

### 2.1 The Furigana & Validation Engine (`KANJI_DICT` in `data.js`)
* **Length-First Matching Algorithm:** The dictionary `KANJI_DICT` uses a specific sorting logic: `Object.keys(KANJI_DICT).sort((a, b) => b.length - a.length)`. This ensures that compound words (e.g., "学生食堂") are matched before single Kanji (e.g., "学"). **Never alter this sorting logic.**
* **Phonetic Variations (濁音/促音):** Words with special pronunciations (like "1本" -> "いっぽん" vs "3本" -> "さんぼん") are handled by adding the full compound word directly into `KANJI_DICT`. 

### 2.2 Dual Answer Validation (in `App.jsx`)
* The `checkAnswer` function validates user input against TWO versions:
  1. The original array answer from `QUIZ_DATA` (usually containing Kanji).
  2. A pure Hiragana version dynamically generated by `getHiraganaVersion()` using `KANJI_DICT`.
* **Workflow for adding new quizzes:** When adding new words/sentences to `QUIZ_DATA` in `data.js`, **ONLY write the Kanji version** (e.g., `['会社員']`). Do not manually add the pure Hiragana version to the array. Instead, ensure the Kanji mapping exists in `KANJI_DICT`.

## 3. Development Standards
* **Adding Data:** All new vocabulary, grammar, sentences, and chapters MUST be added to `src/data.js`. Do not hardcode them into `App.jsx`.
* **Styling:** Use Tailwind CSS utility classes exclusively. Icons are handled via `lucide-react`.
* **Responses:** When proposing changes, provide clear explanations. If you modify `data.js`, ensure you only output the relevant updated blocks (or tell the user where to insert) to avoid outputting massive arrays unnecessarily.

If you understand these rules, you are ready to assist Ibu!
